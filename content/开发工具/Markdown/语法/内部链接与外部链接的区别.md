## 1. 内部链接 (Internal Link)

内部链接主要用于连接库（Vault）内部的其他笔记或文件。

- **语法**：使用双大括号 `[[文件名]]`（WikiLinks 格式）。
- **使用场景**：链接到自己写的笔记、图片、PDF 等库内附件。
- **核心优势**：
  - **关系图谱**：Obsidian 会自动识别这种连接，并在关系图谱中显示连线。
  - **重命名保护**：如果你修改了被链接笔记的文件名，Obsidian 会自动更新所有指向它的链接，确保链接不失效。
  - **快速补全**：输入 `[[` 后，编辑器会自动弹出文件列表供你选择。

## 2. 外部链接 (External Link)

外部链接用于指向库以外的资源，通常是互联网上的网址。

- **语法**：标准 Markdown 格式 `[显示文字](URL地址)`。
- **使用场景**：链接到 GitHub 仓库、参考资料网站、百度网盘等。
- **核心特点**：
  - **通用性**：这是所有 Markdown 编辑器都支持的标准语法。
  - **无关联性**：Obsidian 不会将外部链接记录在关系图谱中。
  - **脆弱性**：如果目标网页删除了，链接就会变成“死链”，Obsidian 无法感知。

---

## 3. 详细对比表

>[!NOTE]
>| 特性         | 内部链接 `[[ ]]`        | 外部链接 `[ ]( )`       |
| :--------- | :------------------ | :------------------ |
| **链接目标**   | 库内的 `.md` 笔记或附件     | 互联网网址、库外的绝对路径       |
| **重命名同步**  | **支持**（文件名变了，链接自动变） | **不支持**（目标变了就失效）    |
| **关系图谱显示** | **显示**（构成知识图谱）      | **不显示**             |
| **双向链接**   | 支持（可查看反向链接情况）       | 不支持                 |
| **语法标准**   | WikiLinks（现代笔记工具常用） | 标准 Markdown（所有平台通用） |

## 4. 链接别名 (Alias) 语法

有时候，笔记的文件名很长或者为了语句通顺，你不希望在正文中直接显示完整的文件名，这时就需要用到 **别名语法**。

- **内部链接别名**：`[[目标文件名|要在正文中显示的文字]]` 
  - *示例*：[[插入标注块|标注块笔记]]
  - *效果*：点击产生的链接跳转到“插入标注块”这篇笔记，但在页面上只显示蓝色字体的“插入标注块”。
---

> [!TIP] 最佳实践
>
> - 只要是库内笔记，**优先使用内部链接**，因为它更稳定且能构成知识网络。
> - 合理使用 **别名语法**，让你的笔记读起来更像自然语言，而不是在读文件清单。
